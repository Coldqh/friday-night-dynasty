## Главная правка

Исправлена неправильная модель колледжских составов.

Больше не должно быть:

```text
один колледж: 8 игроков
другой колледж: 25 игроков
третий колледж: 94 игрока
состав из одних WR
```

Теперь каждый колледж получает стартовый состав:

```text
58 игроков
QB/RB/WR/TE/OL/DL/LB/CB/S/K
FR/SO/JR/SR
нормальный возраст
нормальный eligibility
```

## Что исправлено

### 1. Стартовые ростеры колледжей

Добавлены:

- `collegeRosterPlan.ts`
- `generateCollegePlayers.ts`

Каждый колледж создаётся сразу с полноценным составом.

### 2. Авто-ремонт старых кривых сохранений

Если в старом сохранении колледж имеет:

- меньше 50 игроков;
- больше 64 игроков;
- перекос по позиции;
- отсутствие обязательных позиций;

`normalizeWorldState` через `ensureCollegeLayer` пересобирает ростер до нормального состояния.

### 3. Рекрутинг теперь привязан к позициям после выпусков

После выпуска игроков колледжа система считает, какие позиции просели, и внутренне использует это для рекрутинга.

Требования по позициям больше не выводятся в интерфейсе.

### 4. Убраны требования по позициям из UI

Убрано отображение строк вида:

```text
позиции: WR, CB, K
```

из:

- списка колледжей;
- таблицы колледжей;
- профиля колледжа.

### 5. Таблица колледжей оформлена как школьная

Теперь таблицы школы и колледжей имеют одинаковые колонки:

```text
#
команда
п-б
очк+
очк-
разн
```

### 6. Rivalry колледжей

Rivalry остаётся простым и функциональным:

- пары соперников;
- метка `дерби` в календаре;
- запись в профиле колледжа.
